# Medir o tamanho do seu sistema (aberto, sem preço)

> **Transparência onde importa.** A **medição** é aberta — leia o código, rode você mesmo.
> O que fica protegido é a **régua da auditoria** (como encontramos os problemas), não como
> medimos ou cobramos.

## Como usar
```
python medir/medir.py [caminho-do-seu-projeto]
```
Conta só o **código próprio** (pula framework/vendor/gerado — veja e edite `excludes.txt`;
o `vcl` e outros já estão lá). Não calcula preço.

## O fluxo
1. **Meça** (este script) → você vê o tamanho, e exatamente o que foi contado/pulado.
2. **Envie o número** para receber seu **orçamento**: Humberto Spatz — humberto@spatz.com.br.
3. **Aprovado** → você recebe um **token** que desbloqueia a auditoria (via MCP).
4. A **amostra** (audit1, nesta mesma pasta do repositório) é **grátis**.

## Por que a medição é aberta e o preço não é automático
Medir e cobrar não têm segredo — abrir constrói confiança ("posso conferir como sou medido").
O preço é uma conversa: escopo, desconto do 1º projeto, e o que fizer sentido pro seu caso.
E a **régua** (o método) é o que se paga — essa fica protegida.
